# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅         |
| 0.x     | ❌         |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public issue.

Instead:

1. Contact the maintainer privately.
2. Provide a detailed description.
3. Include reproduction steps if possible.
4. Allow reasonable time for investigation and remediation.

---

## Scope

This project may execute administrative commands when using Auto-Fix mode.

Security reports involving:

* Privilege escalation
* Command injection
* Arbitrary code execution
* Unsafe shell execution
* Credential exposure

will receive highest priority.

---

## Responsible Disclosure

Please allow time for investigation before publicly disclosing vulnerabilities.

Thank you for helping improve the security of this project.
