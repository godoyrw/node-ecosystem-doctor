# Contributing

Thank you for your interest in contributing to Node Ecosystem Doctor.

## How to Contribute

### Reporting Bugs

Please create an issue containing:

* Operating System
* Node.js Version
* Shell Version
* Steps to reproduce
* Expected behavior
* Actual behavior

---

### Suggesting Features

Feature requests are welcome.

Please include:

* Use case
* Expected behavior
* Benefits
* Possible implementation details

---

### Development Workflow

Fork the repository:

```bash
git fork
```

Create a feature branch:

```bash
git checkout -b feature/my-feature
```

Commit your changes:

```bash
git commit -m "feat: add new feature"
```

Push to your fork:

```bash
git push origin feature/my-feature
```

Open a Pull Request.

---

## Coding Guidelines

* Keep compatibility with Bash 5+
* Follow ShellCheck recommendations
* Write readable and maintainable code
* Keep functions focused on a single responsibility
* Avoid unnecessary external dependencies

---

## Commit Convention

Examples:

```text
feat: add npm vulnerability scanner
fix: correct cache validation logic
docs: update installation instructions
refactor: simplify score calculation
```
